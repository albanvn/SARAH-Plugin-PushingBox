    Le principe
  • Plugin autonome ou appelé
       - Le plugin permet de rendre la fonction accessible de "l'extérieur". Voir "La requète via URL"
       - Le plugin peux etre utilisé de façon classique grace au XML
    Les propriétés
    La requète URL
  • Les éléments pour l'appel de la fonction:
           - qui : Nom du destinataire du PUSH, doit exister dans la liste definie par les proprieté du plugins
           - titre : Le titre du PUSH
           - message : Le contenu du PUSH
           - http://IP_DU_NODEJS/sarah/PushingBox?qui=XXX&titre=XXX&message=XX

    La structure de JSON:
  • La structure de retour en JSON:
            {
            "code" : "HTTP CODE",
            "message" : "MESSAGE LIEE au code du statut HTTP"
            "signification" : "Signification, explication du message,
            "tts" : "Phrase vocalisé par SARAH"
            }


    Exemple n°1 : XML
  • '< item >' Dis a Thomas '< tag >'out.action.qui='Thomas''< /tag >< tag >out.action.titre='Message de SARAH:'< /tag >' < /item >
  • La requète URL:
    • http://IP_DU_NODEJS/sarah/PushingBox?qui=Thomas&titre=Message+de+SARAH:&message=dictation

    Tutorial pour PushingBox.com
    1 - https://www.pushingbox.com/index.php
    2 - Active le compte grace a un compte Google
    3 - Ajouter un service suivant votre peripherique. iOS, Android, WindowsPhone ... https://www.pushingbox.com/services.php
    4 - Creer un scenario
     - Remplire le nom du scenario. Par exemple" Plugin PushingBox"
     - Recupere le "DeviceID" pour le plugin
     - Puis "Add in action"
     - Selectionner le peripherique d'affichage du push
     - Dans le POP-UP le "Title" mettre : $titre$
     - Dans "Message" mettre : $msg$
    5 - Dans les proprietes du plugin copier votre DeviceID
